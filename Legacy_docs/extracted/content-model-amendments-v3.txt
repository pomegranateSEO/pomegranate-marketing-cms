  1.   Summary  of  Agreed  Decisions     
# Decision Detail 
D1 
associates  =  brands/partners/software  only 
The  associates  table  is  for  external  brands,  partner  organisations,  and  software  platforms  (e.g.  WordPress,  Google,  SemRush).  It  is  NOT  for  human  authors  or  team  members.  Any  previous  references  to  associates  as  an  author  source  are  superseded. 
D2 
New  public.people  table Create  a  dedicated  people  table  for  authors,  reviewers,  and  team  members.  This  table  is  the  source  of  Person  schema  for  BlogPosting  and  the  team  section  on  the  About  page. 
D3 
blog_posts  author  +  reviewer  FKs 
Add  blog_posts.author_person_id  and  blog_posts.reviewer_person_id  as  nullable  FK  columns  pointing  to  people.id.  Legacy  author_name  /  author_url  /  author_image_url  are  retained  as  transitional  fallback  fields. 
D4 
faq_list  (jsonb)  =  single  FAQ  standard 
The  column  name  faq_list  (jsonb  array  of  objects)  is  the  single  standard  across  pages,  blog_posts,  services,  industries,  free_tools.  Replaces:  pages.faqs,  blog_posts."FAQs",  pseo_page_instances.unique_faqs.  Backfill  then  drop  legacy  columns. 
D5 
schema_json_ld  (jsonb)  =  single  schema  cache  standard 
The  column  name  schema_json_ld  (jsonb)  is  the  single  standard  across  all  content  tables.  Replaces  generated_schema_markup  ARRAY  columns.  pseo_page_instances  already  has  this  column  —  remove  generated_schema_markup  there  once  backfill  confirmed. 
D6 
industries  columns industries  must  include  canonical_url,  keyword_cycling_blocks,  faq_list,  schema_json_ld.  seo_title  and  seo_meta_desc  already  present. 
D7 
free_tools  columns free_tools  must  include  seo_title,  seo_meta_desc,  canonical_url,  keyword_cycling_blocks,  faq_list,  schema_json_ld. 
D8 
businesses  =  default  publisher 
businesses  remains  the  default  publisher  for  all  blog  content.  people  provides  the  Person  author  data.  associates  supports  brand  /  sponsor  /  provider  roles  where  appropriate.
